# `cmd/worker`

这个目录是后台 worker 进程入口。

## 作用

- 轮询或消费待执行任务
- 运行异步任务
- 处理定时任务
- 与 API 进程解耦长耗时执行链路

## 当前文件

- `main.go`

## 当前实现状态

当前版本是最小可运行骨架：

- 周期性调用 `RunDueSchedules()`
- 用于承接未来的异步任务执行器

现在的任务执行主链路仍然主要在 `internal/app` 中，worker 只是独立入口的预留。

## 后续演进

典型演进方向：

1. 接入真正的任务队列
2. 从 API 进程剥离长任务执行
3. 增加重试、退避、幂等和死信处理
4. 接入 PostgreSQL/Redis 后实现跨进程共享状态
