# `internal/domain`

这个目录定义平台的核心领域模型。

## 作用

这里放的是“系统在讨论什么对象”。

当前项目里的领域对象主要有：

- `auth`：用户、登录会话
- `workspace`：工作区与成员关系
- `agent`：Agent 和版本快照
- `tool`：工具定义与调用描述
- `session`：会话与消息
- `task`：任务、步骤、调度
- `approval`：审批对象
- `audit`：审计事件
- `shared`：共享 ID 和密码工具

## 这里不该放什么

原则上不应该放：

- HTTP 处理逻辑
- WebSocket 编码细节
- 数据库存取实现
- 进程启动代码

## 当前状态

当前 `domain` 主要是类型定义，还没有充分发展成包含复杂业务规则的“富领域模型”。

这是因为项目还在首版骨架阶段，先把数据结构和流程打通。

## 后续演进

后续如果领域规则变复杂，可以把一部分规则沉到这里，例如：

- 任务状态迁移约束
- 版本发布规则
- 审批流转规则
- Tool 风险级别判定
- Workspace 角色权限判断
