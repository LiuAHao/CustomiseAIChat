# tests

这个目录用于放置工程化测试，而不是业务源码旁边的包内测试。

当前约定：

- `tests/unit/`：跨包但仍然偏业务逻辑的单元测试说明或未来聚合测试
- `tests/integration/`：接口级、HTTP 级、进程内集成测试
- `tests/fixtures/`：未来放测试样例数据、JSON 请求体、文档样本

当前项目仍保留包内测试，例如：

- `internal/app/app_test.go`

这是合理的，因为应用层测试更适合跟被测包放在一起。

工程化测试目录的意义在于：

- 给后续集成测试留固定位置
- 避免所有测试都堆在 `scripts/`
- 让测试分层和 CI 更容易组织
